# Frontend Redesign Baseline Reset Design

**日期：** 2026-03-18

> 本文档取代 2026-03-17 的前端重设计视觉方向，用于重新定义 Codex Pool 的前端设计基线。

## 背景

上一版重设计虽然解决了“过于默认模板化”的问题，但没有解决更关键的问题：

- 它仍然在借用一整套通用的 AI SaaS 高级感语汇。
- 它的质感更多来自发光、漂浮、雾化、径向渐变和大圆角，而不是来自产品自身的结构、排版和界面判断力。
- 它让高频控制台页面看起来像“被精修过的演示稿”，而不是“被长期使用的专业工具”。

换句话说，上一版不是太旧，而是太像当下流行的生成式后台模板。

## 现状问题

### 1. 背景语言太像模板

- `ParallaxBackground` 的鼠标跟随光斑、模糊大色块、网格洗底，会立刻制造“AI 产品页”联想。
- 登录页与工作台都在共用类似的雾化背景语言，导致产品缺少页面分型差异。

### 2. 表面材质太会表演

- `page-stage-surface`、`page-panel-surface`、sidebar header 都在使用高光边、漂浮阴影、径向亮斑与半透明白膜。
- 页面像一层层“会发光的卡片”叠在背景上，而不是安静地组织信息。

### 3. 排版在模仿“高级感”

- 大写 eyebrow、过紧 tracking、超大 clamp 标题、几乎每个标题都在追求姿态感。
- 结果是页面有设计痕迹，但缺少“编辑判断”和“界面分寸”。

### 4. 控件和数据容器都在轻微悬浮

- 按钮 hover 上抬、输入框重阴影、表头渐变、卡片统一大圆角。
- 单个元素看还行，但全站一起出现，会形成明显的模板复读感。

## 目标

1. 重新建立一套更像“专业器件”而不是“AI 模板”的控制台视觉基线。
2. 保留精致感，但让精致来自边界、比例、留白、排版和材质控制，而不是来自特效。
3. 让 auth、dashboard、workspace、settings 有清楚分型，但共享同一种成熟气质。
4. 让后续页面 rollout 有一条长期稳定、不容易再次漂向模板化的设计准绳。

## 非目标

- 不做营销站风格改造。
- 不做大面积品牌舞台化视觉。
- 不把高频页面改成冷硬工业监控屏。
- 不引入新的 UI 框架替换 React + Tailwind + shadcn 基础。

## 方案对比

### 方案 A：精密工业感

特点：

- 更理性、更硬朗
- 更像专业设备与研究软件
- 几乎完全去掉情绪化材质

优点：

- 最能去 AI 味
- 最适合高频工作台

缺点：

- 做不好会偏冷
- 艺术感不足，容易失去产品自己的脸

### 方案 B：器物感静奢

特点：

- 更温润、更有器件和材质感
- 通过细腻边界、雾面表面、安静色温建立高级感
- 允许少量作者性排版，但不靠舞台化效果

优点：

- 更符合“高端精致、可信克制、专业清晰”
- 比纯工业感更有记忆点

缺点：

- 如果控制不好，会滑向高端品牌站或 lifestyle 视觉

### 方案 C：安静桌面感

特点：

- 接近优秀原生桌面工具
- 极度克制、极低装饰

优点：

- 最耐用
- 风险最低

缺点：

- 太容易保守
- 难以提供足够的产品识别度

## 结论

采用 **方案 B：器物感静奢·工作台版**。

这不是“温柔的品牌站”，也不是“豪华 SaaS 皮肤”，而是：

- 像一件做工很好的专业器件
- 有材质、有分寸、有呼吸感
- 但不会抢走任务、数据、状态和操作的优先级

可以把它理解为：

**以器物感建立精致度，以工作台纪律约束表达强度。**

## 新的设计北极星

Codex Pool 的前端应该更接近：

- 高级专业仪器的控制界面
- 精修过的桌面软件
- 有编辑判断的基础设施后台

而不是：

- 会发光的 AI SaaS 模板
- 漂浮玻璃卡片后台
- 强装饰、强渐变、强舞台感的生成式页面

关键词：

- 安静
- 准确
- 有器件感
- 细致
- 克制的高级
- 有作者性但不炫技

## 设计基线

### 1. 氛围层

- 删除鼠标跟随光斑、模糊大色块、显性网格洗底、彩色发光背景。
- 全局背景改为更淡的纸感/矿物雾面底，只保留很轻的色温变化和结构纹理。
- 背景的职责从“营造氛围”改为“托住内容”。

### 2. 色彩

- 主中性色不再强调青绿科技感，而改为更安静的骨白、烟灰、石墨、墨蓝灰。
- 强调色收成单一且可信的深海蓝灰或氧化青，仅用于焦点、选中、关键状态。
- 成功、警告、危险继续保持语义清晰，但避免糖果色和发光色。

### 3. 表面与材质

- 面板从“漂浮卡片”改为“安静器件表面”。
- 阴影明显减薄，更像接触阴影，不再像悬浮阴影。
- 边界更明确，但不粗重；允许轻微内高光和色温差，不允许 halo 感。
- 圆角整体收小一档，形成更利落、更成熟的器件语言。

### 4. 排版

- 取消满屏 uppercase eyebrow 和过度用力的标题姿态。
- 标题通过更稳的字重、行长、留白和层级建立存在感，而不是靠极限字号和紧 tracking。
- dashboard 与 auth 可保留少量作者性标题；workspace 与 settings 恢复到更稳定的操作语言。
- 数字、统计值、状态标签的字距与粗细要更像软件界面，而不是海报标题。

### 5. 导航与壳层

- sidebar 应像可靠框体，而不是会发光的展示台。
- logo 允许存在，但不再有 halo、漂浮亮斑和材质秀。
- 顶部操作区、分组标题、选中项要更像“被精确布置的控制器件”。

### 6. 控件

- 按钮默认取消上抬式 hover，改为轻微明度、边界、表面层变化。
- 输入框、选择器、表格工具栏减少大阴影和透明白膜，强化边界与填写稳定感。
- 数据表格减少渐变包装和卡片感，更重视信息密度与扫描效率。

### 7. 动效

- 保留统一动效 preset，但整体再降噪。
- 页面进入和面板 reveal 更短、更稳，不做“柔软漂浮”。
- 所有动效都应像界面反馈，而不是装饰动作。

## 页面分型

### Auth

- 品牌感可以保留，但表单必须重新成为主角。
- 不再使用“强舞台 + 亮卡片 + bullet 列表”这类模板化双栏套路。
- 更适合的方向是：一张强秩序的登录表面，加一段更克制的品牌说明。

### Dashboard

- 允许少量作者性排版，作为全站最有“品牌脸”的工作页。
- 但 KPI、趋势、状态仍是主角，不能被材质和装饰压过。

### Workspace

- 任务页默认降噪。
- 导入、配置、排障、管理页面以操作效率、错误清晰度和状态可追踪性为先。
- 精致感来自布局与界面细节，不来自背景和面板表演。

### Settings / Detail

- 是全站最安静、最稳定的页面类型。
- 用结构、间距、表单节奏建立高级感，而不是靠品牌化视觉。

## 明确删除的旧语言

以下内容不再作为默认语言继续使用：

- 鼠标跟随 parallax 背景
- 大面积模糊光斑
- 明显的网格洗底
- card 上的径向高光秀
- 全站统一大圆角
- hover 上抬按钮
- 过多 uppercase badge
- 表头/侧栏的渐变包装感

## 可保留资产

- `page-archetypes.ts` 的页面分型逻辑
- `PageIntro / PagePanel / WorkspaceShell / ReportShell` 的结构抽象
- 现有 admin / tenant 页面的大部分信息架构和任务顺序

需要重做的是视觉语言，而不是重做整套结构。

## 首轮落地范围

1. 全局背景与 token 重写
2. 共享壳层与导航表面重写
3. typography scale 和标题语气重写
4. auth shell 与 login 重做
5. dashboard 首屏重做
6. import jobs / models / config 的表面与任务区降噪

## 验证标准

- 首屏不再让人联想到“AI 帮你做的 SaaS 模板”。
- 登录、dashboard、workspace、settings 的气质统一，但装饰强度明显分层。
- 用户第一眼注意到的是任务、数据、状态，而不是材质、光效和形状。
- 页面在中文长文案、移动端和 reduced-motion 下仍然稳定成立。

## 结论

这次不是继续“优化上一版”，而是明确换轨。

新的设计基线必须从“生成式高级感”切回“有器物感、有分寸、可长期使用的专业工作台”。只要基线改对，后续页面 rollout 才不会继续带着 AI 模板味前进。
